## 什么是web3？ 
- 2014年以太坊创始人加文伍德提出的web3.0概念
✅ web1.0（只读）：1991年第一个网站出现
静态内容、信息聚合
✅ web2.0（可写）：2004年Facebook出现
用户生成内容、数据储存在中心化服务器
✅ web3.0（拥有）：2008年比特币诞生
使用区块链技术、智能合约、去中心化、不可篡改。
每一条公有链都有自己的生态和经济体系，因此会有自己的加密货币来保障生态的自主运行。
4. 从事web3开发是否违法？
✅ 国内法规三条红线
1. ICO IEO:发币、认筹项目，涉及非法集资
2. 开设在国内的交易所项目：类比开设赌场
3. （云）矿场项目：过渡消耗资源
那为什么很多人从事交易所远程工作还没事儿呢？只要交易所符合所在地的法律法规（比如新加坡），且不在国内运营（说白了就是坑不到咱们老百姓的钱），那就没问题。香港作为Web3试验田也设定了几家交易所。
5. 区块链技术特点及各方向要求
✅ 区块链开发和web2开发不太一样的点就是密码学和智能合约的应用。
[Image]
密码学是区块链底层安全机制的保障，所有的区块链都是基于密码学建设的。其中我们要重点学习公钥密码学、哈希函数。
[Image]
智能合约是一种以代码形式记录和执行的合约，它通过预设的条件自动执行合约条款。它有几个特点：去中心化、无需第三方、无需准入、可组合。以太坊生态下，基本都是用solidity创建运行的。因为前端、后端都要和合约产生交互，因此智能合约几乎是必学内容。

✅ 以以太坊生态为例，Web3开发者通常掌握以下技术栈：
-  智能合约开发
项目角度：熟悉主流的去中心化交易所uniswap, dydx, sushiswap, 1inch 原理及源码，熟悉 Compound, Cure 等项目原理及源码，熟悉 lsd 项目 lido 原理及源码，熟悉重新质押协议 eigenlayer 项目原理及源码，熟悉多重质押协议 LinkLayer 项目原理及源码。
编程语言角度：Solidity

- golang以太坊开发
项目角度：使用Golang开发运行在以太坊上的去中心化应用（dApps），这些应用利用智能合约实现业务逻辑，并与用户界面交云。开发基于以太坊的去中心化金融（DeFi）应用和非同质化代币（NFT）应用，利用Golang的高并发处理能力和快速开发优势。
编程语言角度：gorm、grpc、gin等

- Dapp前端开发
项目角度：DApp开发项目涵盖了从智能合约的编写、部署到前端界面的设计，以及与区块链交互的后端服务等多个方面。例如，使用React、Vue3、Next.js等技术栈构建的DApp去中心化应用开源项目。
编程语言角度：React、Solidity、JavaScript、TypeScript等

## 答疑环节 
1. Web3 的前景怎么样？国外市场机会多不多？相比国内传统 IT，容易找到岗位吗？
1. Web3 是一个高速发展的赛道，尤其在国外，市场需求明显高于国内。
2. 国外的企业更加关注区块链、去中心化技术的落地应用，比如 DeFi、NFT、DAO 等领域，这些方向提供了大量远程岗位。
3. 相比国内传统 IT，就业竞争更小，因为 Web3 开发需要特定的技术栈和思维模式，门槛稍高，这恰恰是优势——只要掌握了相关技能，就能脱颖而出。
2. 对 Web3 不太了解，能不能做个扫盲？哪些适合入手，哪些坑不能踩？远程 Web3 的风险大不大？国内政策对这个行业怎么看？
Web3 的方向很多，初学者可以从智能合约开发、DApp 前端开发、后端区块链交互入手，这些方向需求稳定。
需要注意的风险：
- 业务风险：部分项目涉及高风险，比如未经监管的交易平台或非法用途的合约开发，需尽量避开。
- 远程风险：部分远程 Web3 岗位可能缺乏合同保障，要注意公司背景调查。
国内政策： 虽然区块链技术受到支持，但某些业务如虚拟资产交易仍受严格限制，合规性是重点。
3. Web3 开发要怎么学？学习路线怎么规划？多久能达到入门开发水平？
1. 基础知识：学习区块链原理、共识机制、智能合约等基本概念。
2. 开发语言：掌握 Solidity（智能合约开发）和 JavaScript/TypeScript（前端开发），后端掌握golang。
3. 工具链：熟悉 Hardhat、Foundry、Web3.js 或 Ethers.js 等开发工具。
4. 实战项目：结合社区准备一些企业级项目，比如 Defi 或 NFT marketplace。
5. 学习周期：每天花 2-3 小时，3 个月左右可以达到面试水平。
4. 兼职做 Web3 副业靠谱吗？学习周期和收入情况如何？兼职是不是合法？
兼职 Web3 是非常可行的选择，尤其适合已经有一定技术基础的人。
- 周期：前期学习需要 3-6 个月，接单或找兼职项目可以从简单合约或 DApp 开发开始。很多DAO社区有提供大量兼职岗位。
- 收入：根据任务复杂度不太一样，整体副业的平均收入能到2w左右。
- 合法性：确保参与的项目符合当地法规，避免触碰政策红线。
5. 转型 Web3 做远程工作，该选前端还是后端？如果英语一般，有没有希望找到工作？
如果你有前端开发背景，转型 Web3 前端是最快的选择；
如果擅长后端，学习 Go 或 Rust，可以参与底层开发或链上交互。
英语一般没问题，大部分 Web3 岗位更看重技术能力，但提升英语阅读能力有助于面试和学习开源资源。
6. 我是非全本科，找 Web3 的工作会很难吗？是不是学历门槛很高？
学历在 Web3 领域并非决定性因素，更重要的是技术能力和项目经验。
展示你的开发能力和实际成果（如 GitHub 项目），可以有效弥补学历短板。
很多国外企业更注重能力，而不是学历背景。
7. 没有 Web3 工作经历，怎么进 Web3 公司？靠简历包装吗？英语面试是不是特别重要？
没有工作经验，可以从以下方法切入：
- 参与开源项目：为 Web3 社区的开源项目贡献代码，建立技术影响力。
- 模拟项目经验：用开源代码完成一两个个人项目，在简历中展示这些成果。
- 英语面试：英语沟通能力虽然重要，但更关键的是清晰表达你的思路和项目成果。
8. 满足基本条件后，怎么快速进入 Web3 圈子？有什么高效的方法找到靠谱的 Offer？
加入 Web3 社区或线上论坛（像 Discord），参与技术讨论和贡献开源代码，是进入圈子的最佳方式。
此外，定期浏览招聘平台（如 dejob） Web3 相关职位，可以更快找到机会。
9. 37 岁了还能转 Web3 吗？年龄会是个很大的限制吗？
Web3 的发展周期与技术演进有关，短期内不会消退。
远程办公也成为 Web3 企业的主流工作模式，只要行业继续重视全球化合作，远程工作机会就会存在。
10. Web3 远程工作的日常是怎样的？会不会随时被叫起来工作，24 小时都得待命？
完全可以。
Web3 对年龄没有特别限制，关键是掌握核心技能。
你的行业经验（如系统设计、架构优化）反而是优势，能更快融入复杂的区块链项目。

随着区块链技术的快速发展，智能合约已成为连接区块链功能与实际应用场景的关键技术。本课程旨在培养具备专业智能合约开发能力的技术人才，为区块链领域的发展贡献力量。
岗位职责
- 设计和开发安全的智能合约以执行区块链交易和自动化流程。
- 与团队协作，确保智能合约逻辑满足业务需求。
- 进行彻底的合约测试，包括单元测试和集成测试，以确保合约无漏洞和缺陷。
- 维护和优化现有的智能合约代码库，提高合约效率和可扩展性。
- 持续关注区块链行业发展，适时更新和升级合约以适应新技术和规范。

3. DApp前端开发岗位介绍
DApp前端开发岗位是一个专注于去中心化应用（DApp）用户界面和用户体验的角色。DApp前端开发者需要具备传统前端开发的技能，同时也要理解区块链技术的基本原理和特点。
岗位职责
1. 负责 Dapp 智能合约应用的前端开发和维护
2. 与后端和链上进行交互，完成主要的业务逻辑；
3. 项目官网的前端开发和维护；
4. Web 端，移动端的 UI 适配和交互实现。
5. 具备前端系统架构能力，兼顾工程质量和开发效率。

技术栈&要求
1. 3 年以上前端开发经验；
2. 精通 JavaScript，TypeScript 编程语言；
3. 精通 Vue，React 等前端技术栈；
4. 熟练掌握 Node.js 或其他后端开发技术；
5. 熟悉区块链技术，熟悉智能合约开发；
6. 熟悉 Web3，ethers，MetaMask 等 dapp 开发工具；熟悉 walletconnect 原理及应。
7. 熟悉常见的合约和交互流程
